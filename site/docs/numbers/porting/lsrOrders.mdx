[The /lsrorders endpoint](/apis/numbers#operation/CreateLsrOrder) is used to create, modify, and monitor API requests to remove telephone numbers from the Bandwidth network. This API call is intended for use by carriers that want to automate the "Port-out" process with Bandwidth, rather than manually request the removal or "port-out" of each telephone number via a GUI or a Ticket.

## Create Lsr Order

To create Lsr Order you must make a <Highlight color="#079CEE">POST</Highlight> request to our [/lsrorders API](/apis/numbers#operation/CreateLsrOrder) endpoint. This can be done using tools like Postman or cURL. An `order id` will be auto-generated and provided back.

There is a complete list of payload elements of the POST request in api doc the documentation, but in summary the LSR request includes:
- a list of telephone numbers
- information about the subscriber
- tracking information such as a `PON` and a `Customer Order ID`
- winning carrier information such as the `SPID`
- date information

## Fetch Lsr Order Status

This API call uses an asynchronous model that is common in the Bandwidth Dashboard API, where an "order" is created that is then subsequently used to track the progress of the request to completion.

As times can vary and are not guaranteed Bandwidth recommends configuring your account with a subscription instead of polling the order resource for `<OrderStatus>`. Please follow the [How to setup Notification Webhook](/docs/numbers/webhooks/orderWebhook) guide.

To poll for information about your lsr order you can still use a <Highlight color="#00bf8c">GET</Highlight> request to [/accounts/{accountId}/lsrorders/{orderId} API](/apis/numbers/#operation/ReadLsrOrder). This can be done using tools like Postman or cURL.

<Highlight color="#00bf8c">GET</Highlight> <code>https://dashboard.bandwidth.com/api/accounts/{accountId}/lsrorders/{orderId}</code>
<Tabs
    groupId="getLsr"
    defaultValue="curl"

        { label: 'cURL', value: 'curl', },

<TabItem value="curl">

    ```cURL
    curl 'https://dashboard.bandwidth.com/api/accounts/{BW_ACCOUNT_ID}/lsrorders/{orderId}' \
    -X GET \
    -U '{BANDWIDTH_USERNAME}:{BANDWIDTH_PASSWORD}' \
    -H 'Content-Type: application/xml' \
    -H 'Accept: application/xml'
    ```

</TabItem>

## Update Lsr Order

To update port-in request, you must make a <Highlight color="#9a59c5">PUT</Highlight> request to our [/lsrorders/{orderId} API](/apis/numbers/#operation/UpdateLsrOrder) endpoint. This can be done using tools like Postman or cURL.
Modifications are only allowed for orders that are not yet `complete` or `cancelled`.

## Cancel a Port-in Order

PUT is also used to cancel an order, by changing the order status field to `CANCELLED`.  This is the only case where the status can be changed, and when this is done, all other fields are left as they were prior to the cancellation.

### Examples

<Highlight color="#9a59c5">PUT</Highlight> <code>https://dashboard.bandwidth.com/api/accounts/{accountId}/lsrorders/{orderId}</code>
